# TEST_PLAN.md — 手動テスト手順

## スモークテスト(5分・毎変更後に必須)
1. index.html を開く → 5タブが表示され、コンソールエラーなし
2. 在庫: FAB+ → 「鶏もも肉/300g/冷蔵/期限=明日」保存 → 期限が近い食材に「あと1日」で出る
3. 献立: 提案 → 3品出る。鶏もも系レシピが上位に来やすい(期限ボーナス)
4. 「足りない食材を買い物リストへ」→ 買い物タブに遷移し追加されている
5. 買い物: 1品チェック → 「購入済みを在庫へ」→ 在庫タブに増えている
6. レシート: 画像を1枚アップ → サムネイル表示 → タップで拡大
7. 読み上げ: 献立読み上げボタン → 音声が出る(ミュート解除確認)
8. リロード → 全データが残っている

## 回帰テスト(リリース前)
- 在庫の編集・削除 / 期限なし食材が一覧末尾に来る
- レシピ追加(材料改行パース)・編集・削除、seedレシピに削除ボタンが無い
- お気に入り/頻度/評価の保存、「あまり」設定が提案に出にくい
- 「この献立を作った」→ 最終調理日更新 → 直後の再提案で同じ主菜が出にくい
- 買い物曜日トグル、買い物記録+1、エクスポート→全削除→インポートで復元
- スマホ実機: 375px幅でレイアウト崩れなし、カメラ起動、ホーム画面追加

## 家族同期テスト(v1.1)
1. 端末A: 設定 > 家族で同期 > あいことば生成 > 同期を開始 → 🟢になる
2. 端末B: 同じあいことばを入力 > 同期を開始 → 確認ダイアログ → 端末Aの在庫が現れる
3. 端末Bで買い物リストにチェック → 端末Aをバックグラウンド→復帰 → 反映されている
4. 機内モードで在庫追加 → 🔴表示 → 通信復帰後、次の変更操作で🟢に戻る
5. 同期を解除 → ローカルデータは残る/以後通信しない(DevToolsのNetworkで確認)
6. エクスポートJSONに householdId が含まれていないこと
7. レシート画像が他端末に現れないこと(仕様どおり)

## v1.2 機能テスト
1. 献立提案の料理名タップ → 材料と作り方(番号付き)がモーダル表示される
2. 👍また食べたい → freq='よく'/rating+1、👎イマイチ → freq='あまり'(再提案で出にくくなる)
3. 買い物曜日を水・日に設定 → 「まとめて提案」→ 次の買い物日までの日数分が出る(曜日未設定なら3日分)
4. 仕込みリストに にんじん等が合算表示され、明日までの分=冷蔵/3日目以降を含む=冷凍案内になる
5. 仕込みチェックがリロード後も保持される
6. 「まとめ買いリストへ」→ 重複なく買い物リストに入る

## v1.3 機能テスト
1. レシピの間隔ボタン(3日〜1年)を変更→「◯日に1回」表示が変わり保存される
2. 「作った」直後の再提案で同じ料理が出にくく、間隔経過後に出やすくなる
3. 👍で間隔が1段短く、👎で1段長くなる(トーストに日数表示)
4. v1.2のデータで開いても旧freqが自動変換されてエラーが出ない

## v1.4 レシートOCR(Gemini)テスト
1. キー未設定でレシートの「🔍 読み取り」→「先に設定タブでGemini APIキーを登録してなの」トースト+設定タブへ自動遷移
2. 設定タブでAPIキーを保存→レシートタブで「🔍 読み取り」→スピナー表示→確認モーダルが開き、抽出品目がチェックON状態で並ぶ
3. 確認モーダルで1品だけチェックを残し、name/qty/unit/location/expiryを編集→「チェックした1品を在庫に追加」→在庫タブに反映され、チェックを外した品は追加されない
4. 0品抽出または解析失敗時は「読み取れなかったの…手動で追加してなの」と表示され、追加ボタンが出ない
5. APIエラー時: 429→「無料枠の上限。1分待ってなの」、401/403→「APIキーを確認してなの」がトースト表示される
6. エクスポートJSONに Gemini APIキーが含まれていないこと

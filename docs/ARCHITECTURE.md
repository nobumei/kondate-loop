| タブ | 画面 | 主要要素 |

| 🍽 献立 | 今日の提案 / レシピ一覧 | 主菜+副菜+汁物、✓✕材料チップ、検索リンク、⭐頻度評価 |

## ユーザーフロー(コアループ)

1. 在庫タブ: 冷蔵庫の中身を登録(FAB +)
2. 献立タブ: 「献立を提案」→ 期限近い食材優先の3品構成
3. 「足りない食材を買い物リストへ」→ 買い物タブへ自動遷移
4. スーパーでチェック → 「購入済みを在庫へ」
5. レシートタブ: レシート撮影 → 見ながら期限を手動補正
6. 「この献立を作った」→ 最終調理日更新 → 翌日の提案に反映

## 拡張ポイント(コードを触る場所)

- OCR: `runOcr()`(Gemini呼び出し)→ `openOcrModal()`(確認・補正)→ `addOcrItemsToStock()`(在庫反映)。モデル変更は`GEMINI_MODEL`定数のみ
- 外部API: `searchLinks()` を差し替え
- 家族共有: `save()/load()` をリモート同期版に差し替え(スキーマは維持)

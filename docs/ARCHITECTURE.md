```
┌─ index.html (単一ファイル) ─────────────────┐
│ CSS(:rootトークン) + HTML(4タブ+☰設定+モーダル)│
│ Vanilla JS                                  │
│  ├ state管理  load()/save()/defaultState()  │
│  ├ 在庫       renderStock/saveStock...      │
│  ├ レシート    compressImage/handleFiles     │
│  ├ 献立提案    scoreRecipe/pickBest/suggest  │
│  ├ レシピ      renderRecipes/saveRecipe      │
│  ├ 買い物      renderShopping/checkedToStock │
│  ├ 店舗/設定   renderStores/renderDays       │
│  └ 読み上げ    speak* (Web Speech API)       │
└──────────── localStorage (kondate-loop-v1) ─┘
```

## ユーザーフロー(コアループ)

1. 在庫タブ: 冷蔵庫の中身を登録(FAB +)
2. 献立タブ: 「献立を提案」→ 期限近い食材優先の3品構成
3. 「足りない食材を買い物リストへ」→ 買い物タブへ自動遷移
4. スーパーでチェック → 「購入済みを在庫へ」
5. レシートタブ: レシート撮影 → 見ながら期限を手動補正
6. 「この献立を作った」→ 最終調理日更新 → 翌日の提案に反映

## 拡張ポイント(コードを触る場所)

- Gemini呼び出しは共通ヘルパ`callGemini(contents, generationConfig)`(v1.13で新設)に一本化。中身は`{SYNC_URL}/functions/v1/gemini`(Supabase Edge Function、`supabase/functions/gemini/index.ts`)へのPOSTで、Geminiキーはこの関数側のsecretとしてのみ保持されクライアントには一切露出しない。429/401・403/500/その他のエラーメッセージもここで共通化
- OCR: `runOcr()`(`callGemini()`呼び出し)→ `openOcrModal()`(確認・補正)→ `addOcrItemsToStock()`(在庫反映+receipt.items/total保存)。モデル変更は`GEMINI_MODEL`定数のみ。読み取り済み(`r.items`あり)レシートは`openReceiptReview()`(画像+内容の閲覧専用モーダル)から`redoOcrFromReview()`で`runOcr()`に戻れる
- 在庫の追加・合算: `addToInventory({name,qty,unit,location,expiry,memo,addedAt})`が3経路(手動追加・OCR・買い物→在庫)共通の入口。`mergeKey()`(normalizeName+unit+location)が一致する既存行があれば加算、無ければ新規追加。既存データの一括統合は`mergeDuplicateStock()`
- チラシ特売OCR: `runDealsOcr()`(`callGemini()`呼び出し、画像は保存しない)→ `openDealsModal()`(確認)→ `addDealsToState()`(state.deals反映)。スコア加点は`scoreRecipe()`内、バッジ表示は`dealBadge()`/`ingChips()`
- レシピOCR: `runRecipeOcr()`(`callGemini()`呼び出し、画像は保存しない)→ `openRecipeOcrModal()`/`renderRecipeOcrModal()`(レシピごとに材料・作り方を個別編集できる確認モーダル)→ `addRecipeOcrItemsToState()`(チェックした分だけseed:falseのRecipeとしてstate.recipesに追加)
- 外部API: `searchLinks()` を差し替え(現在はクラシル/DELISH KITCHENのサイト内検索へ直リンク)
- 家族共有: `save()/load()` をリモート同期版に差し替え(スキーマは維持)
- 2026-07-13: 家族同期のSupabaseはRLS無効で匿名から全読み書き可の状態だった。householdsを直接アクセス禁止(RLS)にし、あいことばの1行だけ読み書きするSECURITY DEFINER RPC(household_pull/push)経由に変更。publishable keyは公開前提のため防御線はRLSのみという前提を明文化。
- 2026-07-16: 定番レシピをGemini生成で拡充。SEED_RECIPESを18→94種(主菜53/副菜24/汁物17、和洋中+朝食)に。提案のワンパターン解消のため。生成はEdge Function経由でオフライン実行し検証後にベイク。
- 公開レシピプール: `poolFetchTop()`(人気順取得)→ `openPoolModal()`/`renderPoolModal()`(一覧+検索+重複表示)→ `importPoolRecipe()`(ローカルへ取り込み)。貢献は`poolContribute(recipe,source)`(`saveRecipe()`の新規保存時のみ`source='user'`で呼ぶ。ベストエフォート・失敗を握りつぶす)、通報は`poolFlag(id)`/`flagPoolRecipe(id)`。テーブル定義・RPCは`supabase/recipes_pool.sql`
- 2026-07-16: 提案を常時Gemini化(v1.18)。要望が空でも在庫・期限・直近履歴を渡して多様な献立を考えさせ、失敗時は端末内スコアリングにフォールバック。マイクはホールド録音(押下中のみON)、献立クリアで最上部へスクロール、レシピ一覧は在庫一致率順に並べる。
